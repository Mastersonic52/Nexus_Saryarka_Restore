# Протокол №0: Создание изоляционной капсулы (venv)

Прежде чем вводить код, мы должны создать для нашей системы стерильную среду. В мире Python это называется **виртуальным окружением**.

> Чистый `venv` (Virtual Environment) — это «золотой стандарт» индустрии. Его использование учит гигиене разработки и избавляет от классического кошмара: *«А у меня на компьютере эта библиотека уже была установлена, почему у соседа не работает?»*.


### Почему это важно?
Представьте, что Python — это операционная система Нексуса, а библиотеки — это внешние модули. Если один студент установит версию модуля 1.0, а другой 2.0, система может войти в конфликт. `venv` создает копию Python прямо в папке вашего проекта, отделяя её от остального компьютера.

### Как развернуть капсулу в PyCharm 2026.1:

1.  **Создание проекта:** При создании нового проекта выберите **"Custom Environment"** -> **"Generate New"** -> **"Virtualenv"**.
2.  **Путь:** Убедитесь, что папка окружения называется `.venv` и находится внутри папки проекта.
3.  **Активация:** PyCharm 2026.1 сделает это автоматически. Вы увидите название `(.venv)` в нижней части окна терминала.

---

### Ручной запуск (если что-то пошло не так)

Если терминал «молчит» и не видит окружение, используйте команды ручного управления терминалом «Нексуса»:

**Для Windows:**
```bash
python -m venv .venv
.venv\Scripts\activate
```

**Для macOS (M4) / Linux:**
```bash
python3 -m venv .venv
source .venv/bin/activate
```

## Установка модулей связи (Rich)

Теперь, когда капсула активна, установим нашу главную визуальную библиотеку. Введите в терминале PyCharm:

```bash
pip install rich
```

> **Совет для студентов:** Если вы видите желтое предупреждение о том, что `pip` устарел — не паникуйте. Это просто предложение обновить сам «установщик». Для нашей миссии текущей версии будет достаточно.

### Чек-лист готовности среды:

Перед тем как разрешить запустить первый блок кода в Notebook проверьте:
* **Индикатор интерпретатора:** В правом нижнем углу PyCharm должно быть написано `Python 3.1x (.venv)`.
* **Отсутствие красного подчеркивания:** В Блоке №2 строка `from rich import print` не должна быть подчеркнута красным.
* **Папка .venv:** В дереве файлов проекта (слева) должна появиться папка `.venv`. Напомните студентам: **никогда** не пишите в ней свой код, это «технический отсек» системы.

### Небольшая хитрость 2026 года:
В PyCharm 2026.1 есть функция **"Environment Lock"**. Если вы нажмете на иконку замочка в настройках окружения, IDE не позволит случайно установить лишние пакеты, которые могут сломать логику.
